# Práctica 2 - Conceptos Báscos de Base De Datos

# 1. ¿Qué es una Base de Datos Relacional?

Una base de datos relacional es un sistema que permite almacenar grandes cantidades de información.
La información se organiza en tablas compuestas por filas y columnas.
Cada tabla representa una entidad específica del mundo real.
Las tablas pueden relacionarse entre sí mediante claves.
Este modelo reduce la redundancia de datos.
También mejora la integridad y consistencia de la información.
Se administra principalmente usando el lenguaje SQL.

# 2. ¿Qué es una Tabla en el contexto de Bases de Datos Relacionales?

Una tabla es una estructura fundamental dentro de una base de datos relacional.
Está formada por columnas que definen los atributos.
Las filas almacenan los registros individuales.
Cada tabla representa una entidad como alumnos, empleados o productos.
Las tablas facilitan la organización de la información.
Permiten realizar búsquedas y consultas eficientes.
Son esenciales para el correcto diseño de la base de datos.

# 3. ¿Qué es una relación en el contexto de Bases de Datos Relacionales?

Una relación es la conexión lógica entre dos o más tablas.
Permite vincular datos que están almacenados en diferentes tablas.
Se establece usando claves primarias y claves foráneas.
Las relaciones ayudan a evitar la duplicación de información.
También garantizan la integridad referencial.
Gracias a las relaciones se pueden hacer consultas complejas.
Son la base del modelo relacional.

# 4. ¿Qué representa una columna en el contexto de Bases de Datos Relacionales?

Una columna define un atributo específico de una entidad.
Cada columna tiene un nombre y un tipo de dato.
Indica qué tipo de información se puede almacenar.
Por ejemplo, texto, números o fechas.
Todas las filas deben respetar la estructura de las columnas.
Las columnas ayudan a clasificar y organizar los datos.
Son clave para la validación de la información.

# 5. ¿Qué representa un renglón en el contexto de Bases de Datos Relacionales?

Un renglón representa un registro completo dentro de una tabla.
Contiene los valores correspondientes a cada columna.
Cada renglón identifica una instancia única de la entidad.
Por ejemplo, un alumno específico en una tabla de alumnos.
Los renglones permiten almacenar múltiples registros similares.
También se conocen como filas o tuplas.
Son la unidad básica de información almacenada.

# 6. ¿Qué es un motor de base de datos relacional?

Un motor de base de datos relacional es el software que gestiona los datos.
Se encarga de almacenar, recuperar y actualizar la información.
Permite la ejecución de consultas SQL.
Controla la seguridad y los permisos de acceso.
Garantiza la integridad y consistencia de los datos.
Optimiza el rendimiento de las consultas.
Ejemplos incluyen MySQL y PostgreSQL.

# 7. ¿Cuáles son los motores de bases de datos relacionales más populares?

Existen varios motores ampliamente usados en la industria.
MySQL es muy utilizado en aplicaciones web.
PostgreSQL destaca por su estabilidad y funciones avanzadas.
Oracle Database es común en grandes corporaciones.
Microsoft SQL Server se usa en entornos empresariales.
MariaDB es una alternativa moderna y compatible con MySQL.
Cada motor tiene características específicas.

# 8. ¿Cuál es el motor de base de datos ligero que no necesita un servidor?

SQLite es un motor de base de datos ligero y eficiente.
No requiere un servidor para su funcionamiento.
Toda la base de datos se guarda en un solo archivo.
Es fácil de implementar y mantener.
Se usa comúnmente en aplicaciones móviles.
También se emplea en programas de escritorio.
Es ideal para proyectos pequeños o medianos.

# 9. ¿Qué significa SQL en el contexto de bases de datos relacionales?

SQL significa Structured Query Language.
Es un lenguaje estándar para manejar bases de datos relacionales.
Permite crear, modificar y eliminar tablas.
Se utiliza para consultar datos de forma precisa.
También sirve para insertar y actualizar información.
Es compatible con la mayoría de los motores de base de datos.
Es fundamental en el desarrollo de sistemas de información.

# 10. ¿Cuáles son los elementos de un sistema de información basado en bases de datos relacionales?

Un sistema de información está compuesto por varios elementos.
La base de datos almacena la información.
El motor de base de datos administra los datos.
El lenguaje SQL permite la interacción con la base de datos.
Las aplicaciones permiten a los usuarios acceder a la información.
El hardware soporta el sistema.
Los usuarios utilizan y mantienen el sistema.
